# SIMetrix UI Popup Handling Skill

When driving the SIMetrix/SIMPLIS environment via the Command Shell using Python (e.g. `simetrix_automation.py`), the execution of some commands may trigger unexpected GUI popup dialogs. These include:
- **Error Dialogs**: Missing files, syntax errors, parsing issues.
- **Confirmation Prompts**: Overwriting files, un-saved changes warnings.
- **Information Dialogs**: License status, simulation completion popups.

## ⚠️ The Blocking Problem
When a modal popup is spawned by SIMetrix, the application often pauses the execution thread. If our Python script is waiting for output to appear in the Command Shell, it will time out or hang infinitely because the software is waiting for a user click. 

## 🛠️ Pywinauto Solution Strategy
To fix this, `simetrix_automation.py` includes built-in popup interception logic via `pywinauto`.

### How it Works (`SimplisAutomator.check_and_dismiss_popups()`)
1. **Window Enumeration**: The application instances all of its top-level windows (`self.app.windows()`).
2. **Exclusion**: We ignore the "Main Window" (which contains the Command Shell) and target only external child windows or standalone dialogs.
3. **Data Extraction**: For logging purposes, the program extracts the text (`win.texts()`) from the dialog so the AI knows exactly *why* the popup appeared.
4. **Auto-Dismissal**: The script automatically attempts to find standard interactive buttons (like `OK`, `Yes`, `确定`, or `Close`) and clicks them. If no explicit button is found, it forces a window close (`win.close()`).

### When it is triggered
During the `send_command()` execution, the script polls the shell for output text. Within that polling loop, it runs `check_and_dismiss_popups()` periodically (e.g., every 0.5 seconds). If a popup is caught during the execution of a command, it will automatically log the contents and dismiss it, thereby unblocking the command.

## 💡 AI Implementation Guidelines
- **Anticipate Popups**: If you send a command like `Run /file unverified_circuit.net`, assume there's a risk of a simulation error popup. It is natively handled by the loop, but your code must account for the command's success/failure.
- **Examine Console output**: When the script runs, pay attention to the `[弹窗拦截]` warnings printed to standard output. These text messages are vital for debugging *why* a SIMetrix command failed.
- **Extend capability**: Should the user encounter a more complex popup (e.g. a "Save As" file browser), you may need to extend `check_and_dismiss_popups` to detect the specific window title, fill inputs, and press "Save".
