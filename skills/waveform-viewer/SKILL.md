# Waveform Viewer — VCD Waveform Visualization

This skill parses VCD files generated by Verilog simulations and renders publication-quality digital waveform diagrams. It is independent of any specific simulator and works with any standard VCD file.

> [!NOTE]
> This skill follows the unified workspace convention defined in [`SHARED_WORKSPACE.md`](file:///c:/skills-main/skills-main/skills/SHARED_WORKSPACE.md). VCD files are read from `sim/`, waveform images are written to `sim/waveforms/`.

## Capabilities

- **1-bit signals** — Rendered as step waveforms (high/low transitions)
- **Multi-bit buses** — Rendered as hex-annotated bands with transition markers
- **Signal filtering** — Select specific signals by name (supports fuzzy match)
- **Time windowing** — Zoom into a specific time range via `--start` / `--end`
- **Dark theme** — Optimized for readability with a Catppuccin-inspired color scheme
- **Multiple formats** — Output as PNG or SVG

## Usage

### Plot all signals from a VCD file

```bash
python scripts/plot_waveform.py --vcd sim/tb_example.vcd
```

Output: `sim/waveforms/tb_example.png`

### Plot specific signals with a time range

```bash
python scripts/plot_waveform.py --vcd sim/tb_example.vcd \
    --signals clk rst_n counter \
    --start 0 --end 500
```

### Custom output path and title

```bash
python scripts/plot_waveform.py --vcd sim/tb_example.vcd \
    --output sim/waveforms/result.svg \
    --title "Counter Verification"
```

### Built-in demo (no VCD file needed)

```bash
python scripts/plot_waveform.py --demo
```

Generates a demo waveform from an embedded 4-bit counter VCD to verify that the tool works.

## Prerequisites

- **Python 3.10+**
- **matplotlib** — `pip install matplotlib`

No third-party VCD libraries are required; the VCD parser is implemented in pure Python.

## VCD File Generation

For this skill to work, testbenches must include VCD dump statements:

```verilog
initial begin
    $dumpfile("sim/tb_module.vcd");
    $dumpvars(0, tb_module);
end
```

> [!IMPORTANT]
> The `modelsim-agent` skill's testbench template already includes these statements. If you're writing testbenches manually, make sure to add them.

## Workspace Convention

- **Reads from**: `sim/*.vcd` (VCD files produced by simulation)
- **Writes to**: `sim/waveforms/*.png` or `*.svg`
- Waveform image paths are recorded in `sim/sim_result.json` by `modelsim-agent` for cross-skill reference.
